# Vinslipp

Releases, cellar and auction insights for wine lovers.

Vinslipp is a native iOS app and supporting infrastructure that helps wine enthusiasts track upcoming Systembolaget releases, manage a personal cellar, follow Bukowskis wine auctions, and read community tasting notes. Data is fetched daily, rated by AI, and served as data from GitHub Pages.

**App Store:** https://apps.apple.com/app/vinslipp/id6758891213
**Website:** https://vinslipp.app

## Features

### Releases
- Daily feed of upcoming Systembolaget wine launches
- AI-powered ratings (1-4 stars) with short explanations, generated by an AI agent using wine guide context
- Sort and filter by date, producer, wine, vintage, region, country, price, type, and rating
- Mini calendar for date-based navigation
- Horizontal swipe views grouped by category (French Red, French White, Italian Red, Italian White)
- Favorites system shared via App Groups
- iPad side-by-side layout with calendar

### Cellar
- Personal wine collection tracker with full metadata (winery, name, vintage, region, country, style, type, price, quantity, drink year, notes, links)
- Visual bottle chart grouped by drink year or vintage, color-coded per year
- Total bottle count and cellar value summary
- Add wines from releases, auctions, or manually
- Move consumed wines to history; restore from history
- Drag-and-drop reordering for history entries
- CSV export and import (Vinslipp format and Vivino format)
- Migration from legacy UserDefaults storage

### Auctions
- Aggregated Bukowskis auction data across 70+ Bordeaux and Burgundy catalogs
- Producer-level statistics: total lots, sold count, sell rate, average estimate, average hammer price, hammer-to-estimate ratio, premium percentage
- Producer info from wine guide reference
- Live auction lot tracking with estimates, hammer prices, and AI ratings
- Category and rating filters
- Add auction lots directly to cellar

### Blog
- Community tasting notes tied to individual wines
- Firebase Authentication with Sign in with Apple
- Firestore-backed posts with AI agent moderation
- Automatic typo correction, spam and off-topic filtering
- Display name customization, one post per user per day

### Notifications
- Daily push notifications at 9 AM CET via Firebase Cloud Messaging
- Topic-based subscriptions: French Red, French White, Italian Red, Italian White
- Notification body includes the top 5 rated wines of the day
- In-app notification center with read/unread tracking and badge count

### Browser Extension
- Chrome extension (Manifest V3) for Systembolaget and Bukowskis websites
- Injects Vinslipp rating badges onto product cards
- Links wines to relevant search and detail pages

## Architecture

```
Systembolaget API                         Bukowskis
       |                                      |
  wine_cellar.py                       auction pipeline
       |                                      |
  releases.json                    wines.json, auction_stats.json
       |                                      |
  rate_wines.py (AI agent)        rate_auction_wines.py (AI agent)
       |                                      |
       +----> GitHub Pages <------------------+
                   |
          vinslipp.app/data/*.json
                   |
         +--------------------+
         |                    |
      iOS App         Browser Extension
         |
    Firebase (Auth, Firestore, FCM)
         |
    Blog posts, push notifications
```

Data flows in one direction: sources are fetched by Python scripts, processed and rated by an AI agent, committed to the repository, built into a Next.js static export, and deployed to GitHub Pages. The iOS app fetches JSON on launch. Blog and notifications use Firebase as a separate channel.

### Dependencies

- **Firebase** (Auth, Firestore, Cloud Messaging, Core) via Swift Package Manager
- **AuthenticationServices** for Sign in with Apple
- **CryptoKit** for nonce generation
- No third-party UI libraries

### App Structure

The app uses a tab-based layout with five tabs managed in `ContentView.swift`:

1. **Releases** - Systembolaget upcoming launches
2. **Cellar** - Personal wine collection
3. **Blog** - Community tasting notes
4. **Auction** - Bukowskis analytics
5. **Profile** - Settings, notifications, account

### Services

- **DataService** fetches remote JSON from `https://vinslipp.app/data/` for releases, auction stats, and live wines. Caches data in memory for the session.
- **CellarService** manages local cellar storage in `Documents/cellar.json`. Handles CRUD, history, CSV export/import, Vivino import, and migration from the legacy UserDefaults format.
- **BlogService** connects to Firestore for blog posts. Uses Firebase Auth for identity. Caches posts for 30 minutes.
- **FavoritesStore** persists favorite wine product numbers in shared UserDefaults (App Group `group.com.nybroans.vinslipp`).
- **NotificationStore** keeps up to 50 local notification records with deduplication and read/unread state.

### Key Components

- **BottleChart** - Bar chart visualization of cellar wines grouped by drink year or vintage, with tap-to-filter
- **MiniCalendar** - Compact date picker for filtering releases by launch date
- **SwipeView** - Horizontal paging carousel for category-based release browsing
- **WineEditSheet** - Form for adding or editing cellar entries
- **SearchBar** - Shared search input used across tabs
- **FilterChip** / **DetailChip** - Reusable filter and detail display components

### Capabilities

- Push notifications (APNs via Firebase Cloud Messaging)
- App Groups (shared data with potential widget)
- Sign in with Apple
- App Shortcuts (quick access to tabs from home screen)
- Dark/light/system theme selection
- iPad support with side-by-side layouts

## Data Pipeline

### Systembolaget Releases

`wine_cellar.py` queries the Systembolaget public API for upcoming wine launches:

- Filters for 750ml bottles with future launch dates
- Extracts producer, wine name, vintage, price, region, country, type, and product links
- Translates Swedish labels to English (wine types, countries)
- Writes `data/releases.json` (all wines) and category-specific `.txt` files

### Wine Ratings

`rate_wines.py` rates each wine 1-4 stars using an AI agent:

- Searches `wine_guide.json` for up to 3 matching producer/region entries as context
- Calls an AI agent via CLI subprocess
- Produces a score and a 5-word reason per wine
- Injects ratings back into the release data

### Auction Data

`scraper.py` collects data from Bukowskis auction catalogs (D001-D074):

- Parses lot HTML for titles, estimates, hammer prices, auction IDs, and vintages
- Handles accent normalization for producer name matching
- Outputs `wines.json` (full historical data) and `live_wines.json` (active lots)

`merge_stats.py` aggregates per-producer statistics including sell rates, average prices, hammer-to-estimate ratios, and premium percentages. Enriches producers with info from the wine guide.

`rate_auction_wines.py` rates active auction lots using an AI agent.

### Blog Moderation

`moderate_blog.py` reads unmoderated Firestore posts and runs them through an AI agent to fix typos, flag spam, and approve or reject content.

## Browser Extension

A Chrome Manifest V3 extension that enhances Systembolaget and Bukowskis pages:

- **Systembolaget**: injects Vinslipp rating badges on wine product cards
- **Bukowskis**: links auction lots to relevant data
- Fetches `releases.json`, `auction_stats.json`, and `live_wines.json` from `https://vinslipp.app/data/` at runtime

## Automation

### Daily Update (local)

`daily_update.sh` runs on a Raspberry Pi to keep data fresh:

```
git pull --rebase
python wine_cellar.py       # Fetch Systembolaget releases
python rate_wines.py         # Rate wines with AI agent
git push                     # Triggers GitHub Pages deploy
```

### GitHub Actions

**Deploy to GitHub Pages** (`update-releases.yml`): triggered on every push to `main`. Builds the Next.js site, copies JSON data files into the output, and deploys to GitHub Pages.

**Send Wine Notification** (`send-notification.yml`): runs daily at 07:00 UTC (09:00 CEST). Checks each category file for today's releases, builds a notification body with the top 5 rated wines, and sends per-topic push notifications via Firebase Cloud Messaging using a service account.

## Setup

### Prerequisites

- Xcode (with iOS 26+ SDK)
- Node.js 20+
- Python 3.9+ with a virtual environment
- Firebase project with Auth, Firestore, and Cloud Messaging configured
- `GoogleService-Info.plist` in the iOS app directory

### iOS App

```bash
cd ios/App
open App.xcodeproj
```

Build and run on a simulator or device. The app fetches data from `vinslipp.app` on launch, so no local data setup is needed for basic use.

### Web

```bash
cd web
npm install
npm run dev
```

For production build:

```bash
npm run build
```

Output goes to `web/out/` as a static export. Copy data files into `web/out/data/` before deploying.

### Scripts

```bash
python -m venv .venv
source .venv/bin/activate
pip install requests beautifulsoup4 google-auth google-cloud-firestore firebase-admin
```

Scripts that use the AI agent require the CLI to be installed and authenticated.

### Environment

- `firebase-sa.json` - Firebase service account key (not committed, referenced by moderation and notification scripts)
- `auction/cookies.txt` - Bukowskis session cookies in Netscape format (not committed, required by auction pipeline)
- `FIREBASE_SERVICE_ACCOUNT_KEY` - GitHub Actions secret for push notifications
